# 文档规范

本文档规范旨在统一 OpenClaw 项目的文档编写标准，确保文档风格一致、易于维护、便于检索。

## 目录

- [文档说明](001-文档说明) - 了解文档中心的使用方式
- [命名规范](002-命名规范) - 文件与目录的命名规则
- [Frontmatter 规范](003-Frontmatter规范) - 文档元数据标准

## 为什么需要文档规范？

1. **统一风格** - 确保所有文档风格一致，降低阅读成本
2. **便于检索** - 规范的命名和元数据便于快速查找
3. **易于维护** - 统一的结构便于后续更新和维护
4. **团队协作** - 降低沟通成本，提升协作效率

## 开始使用

如果你是第一次使用本文档中心，建议从 [文档说明](001-文档说明) 开始，了解基本的使用方式和规范要求。
