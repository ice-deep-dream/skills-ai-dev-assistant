# 命名规范

本文档规定了 OpenClaw 项目中文档、文件和目录的命名规范，确保项目结构清晰、易于维护。

## 目录结构规范

### 顶级目录组织

```
docs/
├── .vitepress/          # VitePress 配置（隐藏目录，不参与排序）
├── index.md             # 站点首页
├── 01-文档规范/         # 两位数字前缀 + 中横线 + 中文名
├── 02-模板中心/
├── 03-前端开发与计划/
├── 04-后端开发与计划/
├── 05-总结复盘/
├── 06-Bug管理/
├── 07-测试管理/
├── 08-部署运维/
└── 09-变更记录/
```

### 目录命名规则

| 规则 | 说明 | 示例 |
|------|------|------|
| 数字前缀 | 两位数字，用于排序 | `01-`、`02-` |
| 分隔符 | 中横线 `-` | `01-文档规范` |
| 命名语言 | 中文，简洁明了 | `文档规范`、`模板中心` |
| 命名风格 | 名词或名词短语 | `前端开发与计划` |

### 目录编号分配

| 编号 | 目录名 | 用途 |
|------|--------|------|
| 01 | 文档规范 | 文档编写规范和标准 |
| 02 | 模板中心 | 各类文档模板 |
| 03 | 前端开发与计划 | 前端开发文档和计划 |
| 04 | 后端开发与计划 | 后端开发文档和计划 |
| 05 | 总结复盘 | 项目总结和复盘 |
| 06 | Bug管理 | Bug 报告和追踪 |
| 07 | 测试管理 | 测试文档和计划 |
| 08 | 部署运维 | 部署和运维文档 |
| 09 | 变更记录 | 变更历史和记录 |

## 文件命名规范

### 文件命名格式

```
[三位数字]-[中文名称].md
```

### 文件命名规则

| 规则 | 说明 | 示例 |
|------|------|------|
| 数字前缀 | 三位数字，用于排序 | `001-`、`002-` |
| 分隔符 | 中横线 `-` | `001-文档说明` |
| 命名语言 | 中文，简洁明了 | `文档说明`、`命名规范` |
| 文件扩展名 | `.md` | `001-文档说明.md` |

### 示例

```
01-文档规范/
├── index.md                    # 目录索引
├── 001-文档说明.md              # 排序第1
├── 002-命名规范.md              # 排序第2
└── 003-Frontmatter规范.md       # 排序第3
```

### 特殊文件

| 文件名 | 用途 | 必需 |
|--------|------|------|
| `index.md` | 目录首页，展示该目录内容概览 | 是 |
| `README.md` | 兼容传统，一般不主动使用 | 否 |

## 内容命名规范

### 标题层级

```markdown
# 一级标题 - 页面主标题（只能有一个）
## 二级标题 - 主要章节
### 三级标题 - 子章节
#### 四级标题 - 详细内容（尽量少用）
```

### 标题命名规则

| 规则 | 说明 |
|------|------|
| 简洁明了 | 标题控制在 10-20 字以内 |
| 名词优先 | 使用名词或名词短语 |
| 动词一致 | 同级标题使用相同动词形式 |
| 层级清晰 | 避免跳级使用标题 |

### 锚点命名

VitePress 会自动根据标题生成锚点，格式为：

```
# 命名规范 → #命名规范
## 目录结构规范 → #目录结构规范
### 目录命名规则 → #目录命名规则
```

手动创建锚点：

```markdown
## 自定义锚点 {#custom-anchor}
```

## 图片和静态资源

### 目录结构

```
docs/
├── public/              # 静态资源根目录
│   ├── images/          # 图片资源
│   │   ├── logo.svg
│   │   └── screenshots/
│   └── files/           # 可下载文件
└── .vitepress/
```

### 引用方式

```markdown
<!-- 引用 public 目录下的资源 -->
![Logo](/images/logo.svg)
[下载文档](/files/guide.pdf)

<!-- 相对路径引用 -->
![截图](./images/screenshot.png)
```

### 命名规则

| 资源类型 | 命名规则 | 示例 |
|----------|----------|------|
| 图片 | `kebab-case` 描述性名称 | `login-page.png`, `dashboard-dark.png` |
| 图标 | `kebab-case` 功能 + `icon` | `user-icon.svg`, `settings-icon.svg` |
| 文档 | `kebab-case` 版本可选 | `api-guide-v2.pdf` |

## 示例总结

### 完整目录示例

```
docs/
├── .vitepress/
│   └── config.ts
├── index.md
├── 01-文档规范/
│   ├── index.md
│   ├── 001-文档说明.md
│   ├── 002-命名规范.md
│   └── 003-Frontmatter规范.md
├── 02-模板中心/
│   ├── index.md
│   ├── 001-前端模块开发模板.md
│   └── 002-后端模块开发模板.md
└── public/
    └── images/
        └── logo.svg
```

### 文件内容示例

```markdown
---
title: 文档标题
description: 文档描述
category: 分类
tags:
  - 标签1
  - 标签2
outline: deep
---

# 一级标题

## 二级标题

内容段落...

### 三级标题

- 列表项 1
- 列表项 2
```

遵循以上命名规范，可以确保 OpenClaw 项目的文档结构清晰、易于维护和扩展。
