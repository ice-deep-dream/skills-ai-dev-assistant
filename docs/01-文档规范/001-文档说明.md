# 文档说明

本文档中心基于 [VitePress](https://vitepress.dev/) 构建，用于统一管理 OpenClaw 项目的文档、规范和模板。

## 文档中心结构

```
docs/
├── .vitepress/          # VitePress 配置
│   └── config.ts        # 站点配置
├── index.md             # 首页
├── 01-文档规范/         # 文档编写规范
├── 02-模板中心/         # 各类模板
├── 03-前端开发与计划/   # 前端开发文档
├── 04-后端开发与计划/   # 后端开发文档
├── 05-总结复盘/         # 项目总结
├── 06-Bug管理/          # Bug 追踪
├── 07-测试管理/         # 测试文档
├── 08-部署运维/         # 部署运维
└── 09-变更记录/         # 变更历史
```

## 文件命名规范

### 目录命名

- 使用数字前缀 + 中横线 + 中文名称
- 例如：`01-文档规范`、`02-模板中心`
- 数字用于排序，中文用于理解

### 文件命名

- 使用三位数字前缀 + 中横线 + 中文名称
- 例如：`001-文档说明.md`、`002-命名规范.md`
- 每个目录从 `001` 开始编号

## Markdown 基础语法

### 标题

```markdown
# 一级标题
## 二级标题
### 三级标题
```

### 列表

```markdown
- 无序列表项 1
- 无序列表项 2
  - 嵌套项

1. 有序列表项 1
2. 有序列表项 2
```

### 代码块

````markdown
```javascript
console.log('Hello, World!')
```
````

### 链接和图片

```markdown
[链接文本](https://example.com)
![图片描述](image.png)
```

## Frontmatter

每个文档文件都应在开头包含 Frontmatter 元数据：

```yaml
---
title: 文档标题
description: 文档描述
category: 分类
tags:
  - 标签1
  - 标签2
outline: deep
---
```

### 字段说明

| 字段 | 说明 | 必需 |
|------|------|------|
| `title` | 文档标题 | 是 |
| `description` | 文档描述，用于 SEO | 是 |
| `category` | 文档分类 | 否 |
| `tags` | 标签列表 | 否 |
| `outline` | 大纲深度 (`deep` 或数字) | 否 |

## 编写建议

### 内容组织

1. **开头清晰** - 第一段应简明扼要地说明文档目的
2. **结构清晰** - 使用标题层级组织内容
3. **要点突出** - 使用列表、加粗等方式突出重点
4. **示例丰富** - 适当添加代码示例和截图

### 语言表达

1. **简洁明了** - 避免冗长，直接表达核心观点
2. **专业准确** - 使用准确的技术术语
3. **客观中立** - 避免主观评价和情绪化表达
4. **面向读者** - 考虑读者的知识背景和需求

### 格式规范

1. **统一风格** - 保持全文格式一致
2. **正确使用标点** - 中英文标点区分
3. **代码格式化** - 代码块指定语言，保持格式正确
4. **链接有效** - 确保所有链接可正常访问

## 本地预览

在提交文档前，建议在本地预览效果：

```bash
# 进入 docs 目录
cd docs

# 安装依赖
npm install

# 启动开发服务器
npm run docs:dev
```

然后访问 `http://localhost:5173` 预览文档效果。
