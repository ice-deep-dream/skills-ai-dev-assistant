## 文档说明

### Bug 文档

**命名格式**：`BUG-001-Bug标题.md`

**内容要求**：
- 问题描述
- 复现步骤
- 预期结果 / 实际结果
- 原因分析
- 解决方案

### 测试文档

**命名格式**：`TEST-001-测试名称.md`

**内容要求**：
- 测试范围
- 测试用例
- 测试结果
- 问题记录

### 总结复盘

**命名格式**：`总结-YYYYMMDD-主题.md`

**内容要求**：
- 项目概况
- 成果总结
- 问题复盘
- 改进建议

### 开发进度

**命名格式**：`进度-YYYYMMDD-内容.md`

**内容要求**：
- 当前进度
- 完成内容
- 遇到问题
- 下一步计划

## 重要提醒

1. **模块开始/结束时自动记录** - skills 自动执行
2. **开发过程中询问用户** - 是否写入项目跟踪
3. **保持文档更新** - 与实际进度同步
4. **定期回顾总结** - 持续改进
