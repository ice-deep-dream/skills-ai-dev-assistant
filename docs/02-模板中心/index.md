# 模板中心

模板中心汇集了 OpenClaw 项目中常用的各类模板，包括开发模板、Bug 报告模板、测试模板等，帮助团队成员快速创建标准化的文档。

- [后端模块开发模板](002-后端模块开发模板) - Node.js + TypeScript 后端开发模板

### 管理模板
- [Bug 模板](003-Bug模板) - Bug 报告和追踪模板

## 使用说明

1. **选择合适的模板** - 根据文档类型选择对应的模板
2. **复制模板内容** - 复制模板内容到新文件
3. **替换占位符** - 将模板中的 `【】` 占位符替换为实际内容
4. **补充详细信息** - 根据要求补充完整的信息
5. **检查格式** - 确保文档格式符合规范

## 模板编写规范

如果你需要创建新的模板，请遵循以下规范：

### 模板结构

```markdown
---
title: 模板名称
description: 模板描述
category: 模板
tags:
  - 标签1
  - 标签2
outline: deep
---

# 【文档标题】

## 概述

【简要描述本文档的目的和内容】

## 模板内容

### 【章节标题】

【章节内容说明】

```
【代码示例或格式示例】
```

## 使用说明

【如何使用此模板的说明】

## 填写示例

| 字段 | 示例值 | 说明 |
|------|--------|------|
| 【字段名】 | 【示例值】 | 【说明】 |
```

### 占位符规范

- 使用 `【】` 包裹占位符，如 `【标题】`、`【描述】`
- 占位符应具有描述性，说明需要填写的内容
- 必要字段标记为 `【字段名】(必填)`
- 可选字段标记为 `【字段名】(可选)`

### 示例完整性

- 提供完整的填写示例
- 使用表格展示字段和示例值
- 包含边界情况和注意事项

## 模板更新

当需要更新模板时：

1. **版本记录** - 在模板底部添加更新记录
2. **兼容性** - 考虑向后兼容性
3. **通知团队** - 在团队频道通知模板更新
4. **更新示例** - 同步更新填写示例
