# 模板中心

> **核心原则**：模板服务于项目，项目优先于模板
> 
> **重要**：模板在 `auto-init` 时会自动扫描项目并填充通用工具信息

## 模板列表

| 模板 | 用途 | 特点 |
|------|------|------|
| [前端开发模板](./前端开发模板.md) | 前端模块开发 | 自动生成API封装、状态管理、通用组件信息 |
| [后端开发模板](./后端开发模板.md) | 后端模块开发 | 自动生成API返回、分页工具、中间件信息 |

## 初始化说明

### auto-init 执行内容

```
1. 创建 docs 目录结构
   ├── 01-模板中心/
   ├── 02-开发计划/
   └── 03-项目跟踪/
       ↓
2. 分析项目技术栈
   - 检测框架（Vue/React/NestJS/Express等）
   - 检测目录结构
   - 检测命名规范
       ↓
3. 扫描项目通用工具
   - API 请求封装
   - 状态管理方案
   - 通用组件列表
   - 工具函数列表
       ↓
4. 填充模板第二节（项目通用工具）
   - 前端：API封装、Store示例、组件列表
   - 后端：API返回封装、分页工具、中间件
       ↓
5. 完成，可开始开发
```

## 使用流程

### 初始化（模块开始/结束）

```
调用 auto-init → 自动创建目录结构 → 自动填充模板通用信息
```

### 日常开发

```
1. 复制对应模板
2. 填写模块概述和开发内容
3. 开发代码（使用模板中的通用工具）
4. 询问用户是否写入项目跟踪文档
```

---

**记住**：
- 初始化时自动填充通用工具信息，无需手动查找
- 开发时直接参考模板中的封装，保持一致性
