## 🤖 AI 助手必读

### 🚨 强制要求（每轮对话必须执行）

**在开始任何开发任务前，必须按顺序执行以下步骤：**

```
第 1 步：新对话/新项目时自动执行 project-init
       ↓
第 2 步：用户提出新模块需求时自动执行 module-start
       ↓
第 3 步：开发过程中按需执行 module-update
       ↓
第 4 步：用户确认完成时执行 module-complete
       ↓
第 5 步：用户明确要求时执行 create-bug-doc/summary-doc/test-doc
```

**这是强制要求，不是可选项！**

### 📋 触发规则

**自动触发（无需用户明确要求）：**
- ✅ `project-init` - 新对话/新项目时自动执行
- ✅ `module-start` - 用户提出新模块开发需求时自动执行

**手动触发（用户确认后执行）：**
- ✅ `module-complete` - 用户确认模块完成时执行

**按需触发（用户明确要求时）：**
- 📝 `create-bug-doc` - 用户说"创建 Bug 文档"
- 📝 `create-summary-doc` - 用户说"创建总结文档"
- 📝 `create-test-doc` - 用户说"创建测试文档"

### 为什么必须调用？

- ✅ 确保文档与代码同步

- ✅ 自动维护开发进度表格

## 📝 文档编写规则

### Frontmatter 必填字段

所有文档必须包含以下 Frontmatter：

```yaml
---
title: 文档标题
description: 文档描述
category: 分类
tags:
  - 标签1
  - 标签2
outline: deep
---
```

### 开发文档建议字段

```yaml
module: 模块名称
status: planning | developing | completed | on-hold
progress: 0-100
priority: low | medium | high | critical
owner: 负责人
created: YYYY-MM-DD
updated: YYYY-MM-DD
```

### 文档命名规范

- **开发文档**: `001-模块名称开发.md`
- **Bug 文档**: `BUG-001-Bug标题.md`
- **总结文档**: `总结-YYYYMMDD-主题.md`
- **测试文档**: `TEST-001-测试名称.md`

## 🚀 快速开始

### 1. 项目初始化（自动执行）

新对话/新项目时，AI 自动执行 `project-init`：
- 创建 docs 目录结构
- 分析项目技术栈
- 扫描项目通用工具
- 填充模板通用工具信息

### 2. 模块启动（自动执行）

用户提出新模块需求时，AI 自动执行 `module-start`：
- 理解需求 → 输出计划 → 用户确认
- 创建开发文档到 `docs/02-开发计划/`
- 更新 VitePress 配置和进度表格

### 3. 开发过程（按需执行）

开发过程中，AI 按需执行 `module-update`：
- 更新文档内容
- 更新进度状态

### 4. 模块完成（用户确认后执行）

用户确认模块完成时，AI 执行 `module-complete`：
- 更新文档状态为 completed
- 更新进度为 100%
- 更新 VitePress 配置和进度表格

### 5. 本地预览文档

```bash
cd docs
npm install
npm run docs:dev
```

访问 http://localhost:5173 预览
