# AI 开发助手

> 辅助 AI 进行开发文档编写的 Skill

## 核心理念

1. **模板服务于项目** - 项目优先于模板
2. **自动填充通用工具** - 初始化时扫描项目代码，自动填充模板
3. **简化结构** - 合并目录，减少文件数量
4. **按需记录** - 默认不编辑文档，用户要求才写入

---

## 使用场景

| 场景 | 操作 | 说明 |
|------|------|------|
| 项目初始化 | auto-init | 新项目启动时执行一次 |
| 模块开始 | 可选 auto-init | 大模块开始时可重新扫描项目 |
| 模块结束 | 可选记录 | 用户要求时记录到项目跟踪 |
| 开发过程 | 不编辑文档 | 默认不编辑，用户要求才写入 |

---

## 核心功能

### 项目初始化 `auto-init`

新项目启动时执行一次：

```
1. 创建 docs 目录结构
   ├── 01-模板中心/
   ├── 02-开发计划/
   └── 03-项目跟踪/
       ↓
2. 分析项目技术栈
   - 检测框架（Vue/React/NestJS/Express等）
   - 检测目录结构
   - 检测命名规范
       ↓
3. 扫描项目通用工具
   - API 请求封装
   - 状态管理方案
   - 通用组件列表
   - 工具函数列表
       ↓
4. 填充模板第二节（项目通用工具）
       ↓
5. 完成
```

## 目录结构

```
docs/
├── .vitepress/          # VitePress 配置
├── 01-模板中心/         # 开发模板 + 通用工具信息
├── 02-开发计划/         # 前后端开发文档（按需写入）
├── 03-项目跟踪/         # Bug/测试/总结/进度（按需写入）
└── index.md
```

---

## 默认行为

| 情况 | 默认行为 | 触发条件 |
|------|---------|---------|
| 开发过程中 | 不编辑功能文档 | 用户要求才写入 |
| 总结/BUG/测试文档 | 不创建 | 用户要求才写入 |
| 关键功能或思路 | 允许反问确认 | AI 判断需要确认时 |

## 文档规范

### Frontmatter 规范

```yaml
---
title: 文档标题
description: 文档描述
category: 分类
tags: [标签]
status: developing
progress: 0-100
owner: 负责人
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

### 命名规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 开发文档 | `001-模块开发.md` | `001-用户管理开发.md` |
| Bug 文档 | `BUG-001-标题.md` | `BUG-001-登录失败.md` |
| 测试文档 | `TEST-001-名称.md` | `TEST-001-用户测试.md` |
| 总结文档 | `总结-YYYYMMDD-主题.md` | `总结-20240115-复盘.md` |
